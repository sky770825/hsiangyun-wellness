# 網站設計與內容檢查報告

以**高端網頁設計**為方向，針對「網頁設計」與「內容／連結」進行全面檢查，並記錄已實施改動與後續建議。

---

## 一、網頁設計

### 1.1 尺寸與比例

| 項目 | 檢查結果 | 已實施／建議 |
|------|----------|----------------|
| **Container 最大寬度** | 原先僅設定 2xl: 1200px，大螢幕較窄 | ✅ 已調整：Tailwind `container` 改為 sm/md/lg/xl/2xl 斷點，2xl 為 1320px，內容區更符合現代版面 |
| **水平內距** | 部分僅用 `px-6`，小螢幕略擠 | ✅ 已統一：導覽、麵包屑、Hero、頁尾改為 `px-4 sm:px-6 lg:px-8`，並新增 utility `.section-container` 供各區塊使用 |
| **區塊垂直間距** | 內頁多用 pt-32 pb-20（Hero）、py-20／py-24（一般區塊） | 已具一致性；建議後續新區塊維持：Hero 區 `pt-32 pb-20`、一般區塊 `py-20` 或 `py-24` |

**後續建議**

- 內頁各 section 可逐步改為使用 `section-container`（或相同 class），與導覽、頁尾視覺對齊。
- 卡片與按鈕比例維持現狀即可；若新增大型 CTA，可考慮 min-height 或 padding 規範，避免過小或過大。

---

### 1.2 字型與排版

| 項目 | 檢查結果 | 已實施／建議 |
|------|----------|----------------|
| **標題階層** | 原先 h1～h6 皆為同一 `font-size`，層次不明顯 | ✅ 已調整：在 `index.css` 為 h1～h6 設定階層（clamp + 不同 line-height），h1 最大、h2/h3 遞減，提升可讀性與層次感 |
| **內文** | body 使用 Noto Serif TC、line-height 2、letter-spacing 0.02em | 維持現狀，符合溫柔、閱讀感定位 |
| **顯示字體** | Cormorant Garamond + Noto Serif TC | 維持現狀，品牌感足夠 |

**後續建議**

- 各頁標題若已自訂 `text-4xl`、`text-3xl` 等，會覆蓋 base 的 h1/h2 設定，無需改動。
- 若未來要支援「後台調整字級」，可將 `--font-size-base`、`--font-size-heading` 與 h1～h3 的 clamp 連動。

### 1.3 排版與版面

| 項目 | 檢查結果 | 已實施／建議 |
|------|----------|----------------|
| **導覽列** | 固定頂部、捲動後有背景與陰影 | 已符合高端常見做法 |
| **麵包屑** | 內頁有麵包屑、置於 container 內 | 已與 container 內距對齊 |
| **卡片樣式** | 使用 card-pearl、陰影、圓角 | 一致且具品牌感 |
| **按鈕** | hero / golden / soft 等變體、hover 動效 | 維持即可 |
| **響應式** | 首頁與內頁有 md/lg 斷點 | 良好；建議新區塊延續「先手機、再平板、再桌面」的撰寫順序 |

**後續建議**

- 可為「區塊標題 + 副標」訂一個共用 pattern（例如標題 `text-3xl`、副標 `text-muted-foreground`），方便未來擴充頁面時保持一致。

## 三、已實施改動摘要

- **設計**
  - Tailwind `container` 斷點與 2xl 寬度調整。
  - 全站 base 標題階層（h1～h6 字級與 line-height）。
  - 導覽、麵包屑、Hero、頁尾及 utility `.section-container` 統一水平內距（px-4 sm:px-6 lg:px-8）。
- **內容／連結**
  - 頁尾補齊「學員故事」「假瘦語言測驗」，並統一「關於我」「五金剛系統」用字。
  - 行動版選單按鈕改為中文 aria-label 與 aria-expanded。
  - Hero 語錄輪播指示點改為「語錄 N」aria-label。

## 四、後續可選優化（非必須）

1. **設計**
   - 內頁各 `<section>` 逐步改為使用 `section-container`，或至少統一同一套 padding class。
   - 訂定「區塊標題 + 副標」的共用元件或 class，方便日後擴充。
2. **內容**
   - 學員故事頁可視需求加入「預約陪跑」或「免費資源」 CTA。
   - 若提供實際 PDF 或訂閱連結，在後台或 config 填寫 `PDF_DOWNLOAD_URL`、`SUBSCRIBE_QUOTES_URL` 即可。
